# MODULE_STATUS.md

## Objetivo
Tener una visión rápida del estado documental y de implementación de cada módulo.

## Estados

### Documentación
- missing
- draft
- ready

### Implementación
- not_started
- in_progress
- implemented
- stabilized

## Regla principal
No se debe iniciar implementación de un módulo cuya documentación no esté en estado `ready`.
